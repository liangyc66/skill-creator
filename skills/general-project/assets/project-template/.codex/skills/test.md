# Skill: test

## Trigger

Use for:
- core logic change
- bug fix validation
- new behavior validation
- regression protection

---

## Goal

Provide the minimum meaningful validation for the current task.

---

## Inputs

- changed behavior
- risk areas
- existing test patterns
- relevant task file

---

## Procedure

1. Identify what must be proven
2. Prefer existing test style and location
3. Add or update targeted tests when practical
4. Validate critical path manually or by test
5. Record what was verified and what was not

---

## Outputs

- validation scope
- tests added/updated
- manual checks performed
- unverified areas

---

## Constraints

- avoid full-suite runs unless justified
- do not create broad test scaffolding for narrow changes
- do not claim coverage beyond what was verified
