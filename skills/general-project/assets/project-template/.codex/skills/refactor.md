# Skill: refactor

## Trigger

Use for:
- reducing complexity
- improving maintainability
- simplifying logic
- extracting duplication with clear benefit

---

## Goal

Improve code structure without changing intended behavior.

---

## Inputs

- target code area
- current pain points
- risk level
- relevant task file

---

## Procedure

1. Define exact refactor objective
2. Confirm intended behavior to preserve
3. Limit scope tightly
4. Split work into reversible steps
5. Apply one step at a time
6. Verify unchanged behavior after each step
7. Stop when objective is met

---

## Outputs

- refactor goal
- scope
- files changed
- preserved behavior summary
- verification performed

---

## Constraints

- no functional changes unless explicitly included
- no wide cleanup wave
- no renaming churn without benefit
