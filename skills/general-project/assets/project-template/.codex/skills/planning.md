# Skill: planning

## Trigger

Use for:
- L2/L3 task creation
- ambiguous implementation path
- multi-agent work
- work likely to span sessions

---

## Goal

Turn a request into a bounded, executable task plan.

---

## Inputs

- user request
- known repo context
- relevant docs/files
- constraints

---

## Procedure

1. Define goal in one sentence
2. Define success condition
3. Define scope and out-of-scope
4. Classify task level (L1/L2/L3)
5. Identify likely files/modules
6. Choose relevant skill(s)
7. Write stepwise plan
8. Record risks and open questions
9. Assign current owner

---

## Outputs

- task file
- scope definition
- execution plan
- owner assignment
- initial risk register

---

## Constraints

- do not write broad vague plans
- do not include speculative subprojects
- keep plan executable within current task boundary
