# Skill: handoff

## Trigger

Use for:
- pausing work
- switching task
- switching owner
- ending session before completion

---

## Goal

Allow another agent or future session to continue with minimal re-reading.

---

## Inputs

- current task file
- recent findings
- files checked/changed
- current blockers

---

## Procedure

1. Summarize current objective
2. State current status
3. Record what is already done
4. Record what remains
5. Record exact next step
6. Record files that matter most
7. Record risks / traps / false leads
8. Update owner and status if needed

---

## Outputs

- concise handoff note
- updated next step
- updated status
- reduced re-read requirement

---

## Constraints

- do not write generic handoff text
- do not omit blockers
- do not leave next action ambiguous
