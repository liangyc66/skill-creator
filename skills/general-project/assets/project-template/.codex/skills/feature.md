# Skill: feature

## Trigger

Use for:
- new capability
- bounded enhancement
- new behavior within existing architecture

---

## Goal

Add the requested capability with minimal architectural expansion.

---

## Inputs

- feature request
- current behavior
- existing module boundaries
- relevant task file

---

## Procedure

1. Restate the target behavior
2. Define in-scope and out-of-scope
3. Find existing reusable components
4. Choose the smallest viable design
5. Implement in logical steps
6. Verify user-visible behavior
7. Record follow-up ideas separately from current scope

---

## Outputs

- feature summary
- files changed
- behavior impact
- verification result
- limitations / follow-ups

---

## Constraints

- avoid speculative extensibility
- avoid over-generalization
- prefer integration with current patterns
